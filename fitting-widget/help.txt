1. Картинки продуктов должны отдавать корректные html заголовки, разрешающие скачивать их ВСЕМ. Либо нужно будет организовать API позоволяющее обрабатывать полученные с клинета картинки и отдвать ссылку по которой будет можно корректно скачать итоговый холст примерки. Настройки сервера можно посмотреть из этой статьи  https://developer.mozilla.org/ru/docs/Web/HTML/CORS_enabled_image

2. Контракт на Json API
Структура объекта товара
{
  "id": "", - идентификатор товара
  "active": false, - для selected товара true, для оставльных всегда false
  "name": "" - наименование товара
  "height": "", - высота. Если высота не указана, то должна приходить либо пустая строка, либо false
  "width": "", - ширина. Если ширина не указана, то должна приходить либо пустая строка, либо false
  "length": "", - длинна. Если длинна не указана, то должна приходить либо пустая строка, либо false
  "image": "", - url адрес основной картинки товара
  "preview": "", - url адрес превьюшки для слайдера
  "link": "" - ссылка на страницу товара
}

Структура объекта картинки интерьера
{
  "id": "1", - идентификатор картинки
  "active": true, - если нужно чтобы конкретный интерьер был выбран. По умолчанию всегда ставим превому
  "image": "", - url адрес большой картинки
  "preview": "" - url адрес превьюшки для слайдера
}

Структура json файла с данными
{
  "selected": { объект товара с данными }, - содержит только один объект товара. Если пользователь не выбрал товар для примерки, selected может быть ПУСТЫМ ОБЪЕКТОМ, ПУСТЫМ МАССИВОМ, FALSE, либо ВООБЩЕ НЕ ПЕРЕДАВАТЬСЯ

  "viewed": [ - массив с объектами типа товар. Может быть либо ПУСТЫМ МАССИВОМ, либо массивом с данными
    { объект товара },
    ...
  ],

  "sketches": [ - массив с объектами типа картинка интерьера. Должен быть всегда с данными. Отсутствие картинок интерьеров не предусмотрено
    { объект картинки интерьера },
    ...
  ],
  "backlink": "" - url адрес сылки для перехода СО СТРАНИЦЫ примерки. Может быть сслыкой на каталог, главну, или предыдущую страницу с которой пришел пользователь.
}
